# Criteris metodològics del treball per reptes

## Justificació del treball per reptes
El model per reptes s’alinea amb la naturalesa aplicada de DWES: l’alumnat no resol exercicis aïllats, sinó problemes tècnics contextualitzats en un entorn backend semblant al professional.

## Principis metodològics
- aprenentatge situat en contextos reals de desenvolupament
- producte incremental amb valor acumulatiu entre reptes
- evidències tècniques verificables per damunt de declaracions teòriques
- iteració curta: planificar, implementar, provar, documentar i defensar
- retroalimentació contínua basada en codi, repositori i execució

## Estructura interna comuna de cada repte
1. Context professional i objectiu operatiu.
2. Requisits tècnics mínims i criteris de qualitat.
3. Activitats guiades de construcció.
4. Activitats d’ampliació i refactorització.
5. Evidències obligatòries (codi, proves, documentació, IA log quan toque).
6. Tancament amb demostració i defensa tècnica.

## Tipus d’activitats
- anàlisi i decisió tècnica (comparativa de solucions)
- implementació incremental de funcionalitats backend
- depuració de casos reals i resolució d’errors
- disseny i execució de proves
- documentació tècnica orientada a manteniment
- revisió de codi i millora sobre feedback

## Entorn professional mínim
- Git i GitHub amb historial de commits significatius
- branques de treball i revisió abans d’integrar
- gestió d’issues i traçabilitat de tasques
- debugging sistemàtic i proves mínimes per repte
- documentació operativa: README, decisions tècniques i API docs quan corresponga

## Paper del professorat
- definir el marc tècnic i les restriccions de cada repte
- modelar criteris professionals de qualitat i verificació
- donar feedback accionable sobre codi, proves i documentació
- verificar autoria, comprensió i ús responsable de la IA

## Paper de l’alumnat
- planificar i executar el repte amb evidències traçables
- justificar decisions tècniques i alternatives descartades
- validar el codi amb proves i depuració pròpia
- documentar l’evolució del producte i defensar el resultat
- usar la IA com a suport, no com a substitut de comprensió
