# Rúbrica base per reptes

## Finalitat de la rúbrica
Disposar d’un instrument comú i reutilitzable per a avaluar tots els reptes del mòdul DWES amb criteris observables, coherents amb els RA i orientats a evidències tècniques autèntiques.

## Dimensions d’avaluació
1. Comprensió tècnica
2. Producte funcional
3. Qualitat del codi
4. Testing i debugging
5. Documentació tècnica
6. Ús de Git i traçabilitat
7. Ús verificable de la IA
8. Defensa tècnica

## Nivells de domini
- **N4 Excel·lent:** domini sòlid, decisions justificades i resultats consistents.
- **N3 Adequat:** resolució correcta amb justificació suficient i errors menors.
- **N2 Bàsic:** assoliment parcial, amb dependència de suport extern i incoherències.
- **N1 Insuficient:** no compleix mínims funcionals o no es pot verificar autoria/comprensió.

## Descriptors observables per dimensió

| Dimensió | N4 Excel·lent | N3 Adequat | N2 Bàsic | N1 Insuficient |
|---|---|---|---|---|
| Comprensió tècnica | Explica decisions, alternatives i impacte amb criteri professional. | Explica la major part de decisions amb alguna limitació de profunditat. | Explicació parcial i poc connectada amb el codi real. | No pot explicar decisions clau o mostra desconeixement crític. |
| Producte funcional | Solució completa, estable i alineada amb l’encàrrec del repte. | Solució funcional amb incidències menors no crítiques. | Solució incompleta o amb errors recurrents de funcionament. | Solució no funcional o no entregada en condicions. |
| Qualitat del codi | Codi clar, modular, consistent i mantenible. | Codi correcte amb alguns punts de millora estructural. | Codi poc consistent, amb deute tècnic notable. | Codi desordenat, difícil de mantenir o incoherent. |
| Testing i debugging | Proves rellevants i registre de depuració amb correccions justificades. | Proves bàsiques i depuració funcional amb evidència parcial. | Proves mínimes insuficients i depuració reactiva. | Sense proves útils ni evidència real de depuració. |
| Documentació tècnica | README/docs completes, executables i alineades amb el projecte. | Documentació suficient per executar i entendre el lliurament. | Documentació incompleta o amb llacunes operatives. | Documentació absent, desactualitzada o no fiable. |
| Ús de Git i traçabilitat | Historial net, commits significatius, branques/issues ben utilitzades. | Historial usable amb traçabilitat general correcta. | Historial poc clar o amb traçabilitat parcial. | Historial caòtic, sense traçabilitat o difícilment verificable. |
| Ús verificable de la IA | IA usada amb criteri, AI log complet i validació tècnica demostrada. | IA usada de forma acceptable amb evidències de validació. | IA usada amb control limitat i justificació insuficient. | Delegació excessiva, incoherències o absència de verificació. |
| Defensa tècnica | Defensa precisa, autònoma i coherent amb evidències del repo. | Defensa correcta amb dubtes puntuals no crítics. | Defensa parcial, amb dependència de guió o suport extern. | No pot defensar el treball ni relacionar-lo amb les evidències. |

## Pes orientatiu per dimensió

| Dimensió | Pes orientatiu |
|---|---|
| Comprensió tècnica | 15% |
| Producte funcional | 20% |
| Qualitat del codi | 15% |
| Testing i debugging | 10% |
| Documentació tècnica | 10% |
| Ús de Git i traçabilitat | 10% |
| Ús verificable de la IA | 10% |
| Defensa tècnica | 10% |

> Els percentatges es poden ajustar lleugerament per repte, però sempre mantenint les 8 dimensions.

## Relació amb evidències
Cada dimensió s’avalua sobre evidències observables del repte:
- codi i estructura del projecte
- historial Git (commits, branques, issues)
- proves i registre de debugging
- documentació tècnica (README, ADR, API docs, etc.)
- AI log i justificació de decisions quan hi ha ús de IA
- defensa tècnica final

## Nota sobre ús verificable de la IA
La IA és una eina admesa i habitual, però la qualificació depén de la verificació:
- l’alumnat ha d’entendre i poder modificar el codi entregat
- tota aportació de IA ha d’estar validada amb proves o execució real
- la coherència entre repositori, documentació i defensa és condició imprescindible
- la delegació excessiva o no verificable penalitza les dimensions de comprensió, IA i defensa
