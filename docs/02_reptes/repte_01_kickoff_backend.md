# Repte 1. Kickoff tècnic d’un servei web backend

## Finalitat
Posar en marxa la base tècnica d’un servei web backend realista i deixar preparat un punt de partida professional per als reptes següents.

## Relació amb resultats d’aprenentatge
- **RA1 (principal):** selecció d’arquitectura i preparació de l’entorn de desenvolupament.
- **RA5 (inici):** primera organització del projecte amb criteris de mantenibilitat.

## Context professional
Un equip de desenvolupament rep l’encàrrec d’arrancar un nou servei backend. Abans d’implementar funcionalitats, necessita establir repositori, estructura base, convencions de treball i una decisió arquitectònica defensable.

## Problema o encàrrec inicial
L’equip ha de lliurar en poc temps un kickoff tècnic que permeta començar a produir codi de manera segura i ordenada. El lliurable ha de demostrar que:
- la base de projecte funciona
- l’arquitectura triada està justificada
- el flux de treball amb Git/GitHub és executable

## Producte final
- repositori inicial operatiu amb estructura clara
- README tècnic de posada en marxa
- ADR (Architecture Decision Record) de la decisió principal
- primera validació tècnica executada i documentada

## Seqüència de treball
1. Revisar requeriments mínims del servei i restriccions del context.
2. Comparar opcions d’stack backend i argumentar una elecció.
3. Configurar entorn local i esquelet inicial del projecte.
4. Definir convencions de repositori (branques, commits, issues).
5. Realitzar una validació tècnica mínima (execució + comprovació bàsica).
6. Documentar decisions i deixar evidències traçables.

## Activitats o microtasques
- crear issue inicial de kickoff i desglossar microtasques
- inicialitzar repositori i estructura de carpetes coherent
- preparar scripts o instruccions bàsiques de posada en marxa
- redactar README tècnic (instal·lació, execució, verificació)
- redactar ADR curt amb alternatives valorades
- registrar la primera prova de funcionament del servei
- preparar mini defensa tècnica (5-7 minuts)

## Materials i apunts associats
- bloc 1 d’arquitectures web
- introducció a entorn de treball backend
- guia de convencions Git/GitHub del curs
- materials bàsics de documentació tècnica (README, ADR)

## Paper de la IA
La IA s’utilitza com a suport per a:
- comparar alternatives d’stack i patrons inicials
- proposar una estructura base de projecte
- millorar claredat de documentació tècnica
- suggerir comprovacions inicials de validació

La decisió final sempre recau en l’alumnat, que ha de verificar i adaptar qualsevol proposta generada.

## Evidències d’aprenentatge
- historial de commits inicials amb missatges significatius
- issue de kickoff i tancament de microtasques
- README tècnic complet i executable
- ADR amb justificació de decisió
- registre de primera validació tècnica (comanda, resultat, incidències)
- AI log quan hi haja suport de IA

## Instruments d’avaluació
- rúbrica base de reptes (adaptada al Repte 1)
- checklist de revisió de repositori
- revisió de README i ADR
- defensa tècnica breu individual o per equip

## Riscos d’ús inadequat de la IA
- acceptar propostes tècniques sense entendre implicacions
- generar documentació aparentment correcta però no executable
- incoherència entre codi, decisions i explicació oral
- delegació de decisions d’arquitectura sense validació pròpia

## Mesures de verificació
- comprovació en directe de posada en marxa del projecte
- preguntes tècniques sobre alternatives descartades
- contrast entre AI log, commits i resultat final
- microcanvis en viu sobre README o configuració
- revisió de traçabilitat de decisions en issue + ADR

## Entorn professional
- Git i GitHub com a base de treball
- ús d’issues per a planificació inicial
- branques i integració controlada de canvis
- debugging inicial de configuració i execució
- documentació tècnica mínima de projecte (README + ADR)

## Definition of done del repte
El Repte 1 es considera completat quan:
- el repositori arranca en local seguint el README sense passos ocults
- existeix una decisió arquitectònica documentada i defensable (ADR)
- hi ha historial de treball traçable (issues + commits)
- s’ha executat una validació tècnica mínima amb resultat verificat
- l’equip/alumnat pot explicar què ha triat, per què i com continuarà en el Repte 2
