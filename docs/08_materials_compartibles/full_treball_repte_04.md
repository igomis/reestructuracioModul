# Full de treball del Repte 4

## Objectiu

Publicar una part rellevant del producte com a `API` usable, provada i documentada.

## Tasca

- tria quin recurs o cas d'ús real exposaràs
- defineix entrada, eixida i error mínim de l'endpoint principal
- implementa almenys una operació de consulta i una d'acció o escriptura
- prova un cas correcte i un cas d'error rellevant
- documenta com es consumix l'`API`
- deixa preparada la base per a `R5`

## Evidència mínima

- contracte mínim d'`API`
- endpoint principal funcional
- prova o col·lecció reproduïble
- consum real des d'una eina o client simple
- documentació breu alineada amb el comportament real

## Validació mínima

- llança una petició correcta i una incorrecta
- mostra el codi d'estat o resposta esperada
- ensenya la prova o col·lecció reutilitzable
- comprova que la documentació coincidix amb el resultat real

## Errors habituals

- obrir rutes sense un cas d'ús clar del producte
- publicar un `CRUD` ornamental
- documentar una cosa i retornar-ne una altra
- provar només el cas feliç
- no deixar cap evidència real de consum

## Ampliació si acabes prompte

- millora tractament d'errors
- reforça documentació o col·lecció de proves
- afegeix una segona operació coherent sobre el mateix recurs
